# SeChatbot 代码公约

> *AI 代码审核的评判依据。所有升级代码必须通过此公约的检查。*

---

## 一、核心原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **资金安全** | 不得重定向、窃取、锁定用户或 Bot 的资金 |
| 2 | **权限不变** | 不得绕过 Sponsor 角色系统或投票治理机制 |
| 3 | **数据主权** | 用户数据归属用户，不得新增隐藏的数据外传 |
| 4 | **可审计性** | 所有状态变更必须有日志，不得有静默操作 |
| 5 | **去中心化** | 不得引入任何单一控制点或后门管理员 |
| 6 | **向后兼容** | 不得破坏现有数据格式和 API 契约 |

---

## 二、AI 审核清单

### 2.1 资金安全 (权重: 35%)

```
□ 是否有新的钱包地址被硬编码？          ← 后门信号
□ 是否有资金转账逻辑被修改？            ← 可能重定向捐赠
□ 是否有新增的收款地址？                ← 可能劫持付款
□ 是否有 gas/手续费相关的异常操作？       ← 可能消耗 Bot 余额
□ 是否修改了 USDT 合约地址？             ← 可能指向假冒合约

★ 任何一项违规 → 自动否决
```

### 2.2 权限与治理 (权重: 25%)

```
□ 是否引入了新的管理员角色？             ← 可能绕过 Sponsor 体系
□ 是否修改了投票逻辑或阈值？             ← 可能操纵治理
□ 是否绕过了代码锁定机制？               ← 可能禁用安全保护
□ 是否新增了不受 Sponsor 控制的接口？    ← 可能开后门

★ 任何一项违规 → 自动否决
```

### 2.3 数据安全 (权重: 20%)

```
□ 是否新增了网络请求到未知服务器？        ← 可能外传数据
□ 是否读取了不应读取的本地文件？          ← 可能窃取密钥
□ 是否修改了日志级别或日志内容过滤？      ← 可能隐藏行为
□ 是否新增了用户数据收集？                ← 可能侵犯隐私

★ 违规 ≥ 2 项 → 自动否决
```

### 2.4 代码质量 (权重: 20%)

```
□ 是否有明显的逻辑错误或死循环？          ← 可能导致 DoS
□ 是否有未处理的异常或静默 try/catch？    ← 可能隐藏错误
□ 是否有硬编码的密钥或 Token？            ← 安全风险
□ 依赖是否有已知漏洞？                    ← 供应链风险

★ 扣分项，不自动否决，但影响总分
```

## 三、评分标准

| 分数 | 判定 | 后续行动 |
|------|------|---------|
| **≥ 90** | ✅ 通过 | 创建投票提案给社区 |
| **70-89** | ⚠️ 需人工复核 | 标记高风险项，仍可发起投票 |
| **< 70** | ❌ 拒绝 | 不创建提案，记录拒绝原因 |
| **任一红线项违规** | 🔴 直接否决 | 不创建提案，记录违规详情 |

### 红线项 (一票否决)

```
1. 新增或修改钱包地址/收款地址
2. 引入后门管理员或硬编码密钥
3. 新增向未知服务器的网络请求
4. 绕过代码锁定或投票机制
5. 修改资金流向逻辑
```
